# RE:CAR 관리자 페이지 UI/UX 구조 설계 리포트

작성일: 2026-05-18

## 1. 운영 목표

관리자 페이지는 통계 화면이 아니라 “오늘 뭘 처리해야 하는지”를 바로 보여주는 CRM이어야 합니다. 접속 직후 미배정, 오늘 연락, 재연락 예정, 타임특가 문의, 진행 중 DB, 부결/장기미응답 DB가 먼저 보여야 상담 실수를 줄일 수 있습니다.

## 2. 최종 화면 구조

### HOME

- 미배정 DB
- 오늘 연락할 DB
- 재연락 예정 DB
- 신규 타임특가 문의
- 내 진행 중 DB
- 부결/장기미응답 DB

HOME은 숫자만 보여주는 대시보드가 아니라, 누르면 바로 해당 DB 카드 목록으로 이동하는 행동형 카드 구조가 맞습니다.

### DB 관리

표는 가로 스크롤과 누락 위험이 커서 카드형으로 전환했습니다.

각 카드에는 고객명, 전화번호, 문의 시간, 상태 배지, 담당자, 문의 유형, 희망차량, 계약유형, 예상 월납입금, 초기비용, 요청사항, 타임특가 가격 정보가 한 화면 안에 정리됩니다.

카드 하단에는 상태 변경, 담당자 배정, 재연락 날짜, 담당자 메모, 오늘 다시 연락, 내일 다시 연락, 전화 클릭, 카카오톡 클릭을 배치했습니다.

## 3. 상태 흐름

최종 상태값은 다음 순서가 실제 영업 흐름에 가장 맞습니다.

`선택 → 미배정 → 부재 → 재연락 예정 → 상담 → 심사 → 발주 → 출고 → 부결`

부재 상태인데 재연락 날짜가 없으면 경고가 보여야 합니다. 재연락 날짜가 오늘이면 HOME의 “오늘 연락할 DB”에 자동 노출되는 구조가 가장 안전합니다.

## 4. 권한 구조

최고관리자는 전체 DB 조회, 담당자 배정, SD/AD 권한 변경, 그룹 생성, CSV 다운로드, DB 삭제, 테스트 DB 분리, 가입 승인/거절이 가능합니다.

SD는 자기 그룹 DB를 조회하고 같은 그룹 AD에게 재배정할 수 있습니다. 다른 그룹 DB와 최고관리자 권한은 건드릴 수 없습니다.

AD는 본인 DB만 확인하고 상태, 메모, 재연락 날짜, 고객명만 수정할 수 있습니다. 전화번호, 담당자, 삭제, CSV는 제한해야 합니다.

## 5. 신뢰와 실수 방지 장치

- 미배정 DB는 항상 상단 우선 노출
- 상태 배지는 색상으로 즉시 구분
- 담당자 변경과 상태 변경은 이력으로 남김
- 전화/카카오톡 클릭 이력 저장
- 중복 전화번호 자동 감지
- 테스트 DB와 운영 DB 분리
- 모바일/태블릿/PC에서 가로 스크롤 없이 카드가 4열 → 2열 → 1열로 접힘

## 6. 운영 기준

신규 문의는 미배정 DB에 먼저 들어오고, 담당자를 배정하면 해당 AD의 “나의 DB”에 나타납니다. 재연락 날짜를 오늘로 지정하면 HOME에 노출됩니다. 출고/부결 DB는 일반 진행 DB와 분리해서 봐야 상담 우선순위가 흐려지지 않습니다.

현재 구조는 예쁜 관리자 페이지보다 실제 영업팀이 놓치지 않고 처리할 수 있는 CRM형 구조를 기준으로 설계했습니다.
