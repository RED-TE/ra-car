# RE:CAR 관리자 페이지 UI/UX 구조 설계 리포트

작성일: 2026-05-18

## 1. 현재 문제

- DB 표가 너무 넓어 가로 스크롤이 생기고, 상담자가 고객명/연락처/차량/메모를 한 번에 보기 어렵습니다.
- 상태 변경, 담당자 배정, 메모 입력이 같은 행 안에 흩어져 있어 업무 우선순위가 바로 보이지 않습니다.
- 테스트 DB와 실제 고객 DB가 섞이면 신뢰도가 떨어지고, 상담 실수가 생길 수 있습니다.
- 최고관리자, SD, AD 권한은 존재하지만 화면 구조상 “내가 지금 봐야 할 DB”가 더 선명하게 구분될 필요가 있습니다.

## 2. 이번 반영 구조

- 가로 스크롤 표 중심 UI를 카드형 DB 리스트로 변경했습니다.
- 각 고객 카드는 `선택 / 고객정보 / 희망차량·요청사항 / 상태·배정·메모` 순서로 읽히게 구성했습니다.
- 담당자 메모 영역을 카드 내부에서 크게 유지해 상담 기록을 바로 남길 수 있게 했습니다.
- 타임특가 문의는 `타임특가 문의` 배지와 가격 정보를 희망차량 아래에 노출하도록 유지했습니다.
- 화면 폭이 좁아지면 카드 내부가 4열 → 2열 → 1열로 자동 접히도록 구성했습니다.

## 3. 권장 정보 구조

### HOME

- 오늘 할 일
- 오늘 일정
- 상태 요약
- 빠른 검색
- 최근 미배정 DB

### 나의 DB

- AD 본인 담당 DB만 표시
- 상태, 메모, 고객명만 수정 가능
- 전화번호는 읽기 전용 유지

### DB 관리

- 최고관리자와 SD용 화면
- 전체 또는 그룹 DB 조회
- 담당자 배정
- 진행상황 일괄 필터
- CSV 다운로드

### 등급 관리

- 최고관리자 전용
- SD/AD 권한 부여
- 그룹 생성 및 배정
- SD는 같은 그룹 AD의 DB 확인 및 재배정 가능

### 가입 신청

- 최고관리자 전용
- 가입 신청 승인/거절
- 승인 후 등급과 그룹 배정

## 4. 운영 UX 원칙

- 상담자는 “지금 연락할 DB”를 먼저 보게 해야 합니다.
- 상태값은 `선택 → 부재 → 상담 → 심사 → 발주 → 출고 → 부결` 흐름으로 유지합니다.
- 미배정 DB는 눈에 띄게 보여야 합니다.
- 메모는 가장 크게 두고, 전화번호는 실수 방지를 위해 AD에게 수정 권한을 주지 않는 편이 안전합니다.
- 테스트/검증 DB는 운영 DB와 절대 섞이지 않게 삭제 또는 별도 개발 프로젝트로 분리하는 것이 좋습니다.

## 5. 다음 개선 제안

- 미배정 DB만 모아보는 상단 고정 패널
- 부재 고객 재연락 날짜 자동 등록
- 상태 변경 시 자동 타임라인 기록
- 카카오톡/전화 클릭 이력 저장
- 중복 전화번호 자동 감지
- 고객 카드에서 바로 `오늘 다시 연락`, `내일 다시 연락` 버튼 제공
